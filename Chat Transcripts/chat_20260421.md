```116:140:src/quantitative_trading/agents/rule_one/agent.py
    def is_buy_quant_random_qual(
        self,
        seed: str,
        base_rate_meaning: float,
        base_rate_moat: float,
        base_rate_management: float,
    ) -> bool:
        """Variant C: quant pass × deterministic-random qualitative draws.

        The randomness is a function of (ticker, as_of, seed) so the variant is
        reproducible across backtest runs. Each of the 3 Ms is drawn
        independently with its given base rate.
        """
        if not self.quant_pass:
            return False
        for m, rate in (
            ("meaning", base_rate_meaning),
            ("moat", base_rate_moat),
            ("management", base_rate_management),
        ):
            key = f"{seed}|{self.ticker}|{self.as_of.isoformat()}|{m}".encode()
            digest = hashlib.sha256(key).digest()
            draw = int.from_bytes(digest[:8], "big") / 2**64
            if draw >= rate:
                return False
        return True
```

## Why the random-qual base rates are calibrated to A

If C just used 50% per M, it'd be far less selective than A and we couldn't compare. So we measure A's actual LLM pass rates, then have C draw with those same rates:

```47:78:src/quantitative_trading/backtest/engine.py
def add_random_qual_decision(
    df: pd.DataFrame,
    *,
    seed: str = "rule-one-ablation-v1",
    decision_col: str = "decision_quant_random_qual_buy",
) -> pd.DataFrame:
    """Add a `decision_quant_random_qual_buy` column to df.

    The synthesized "qualitative pass" for each row is sampled deterministically
    from the variant-A base rates (overall fraction of times each M passed in
    the LLM-evaluated rows). Rows where quant fails are False; otherwise the
    three Ms are drawn independently and ANDed together.
    """
    out = df.copy()

    # Compute LLM base rates only over rows where the LLM was actually run.
    llm_rows = out[out["check_meaning"].notna()]
    if len(llm_rows) == 0:
        out[decision_col] = False
        return out

    base_meaning = float(llm_rows["check_meaning"].astype(bool).mean())
    base_moat = float(llm_rows["check_moat"].astype(bool).mean())
    base_management = float(llm_rows["check_management"].astype(bool).mean())
```

In your smoke build the LLM base rates are: Meaning 83.8%, Moat 71.3%, Management 25.7%. So C draws three coins for each row at exactly those probabilities. That makes A and C **equally selective on average** — any difference in their hit rate must come from the LLM identifying the *right* companies to pass, not just from being more or less picky.

## What each comparison tells you

- **A.precision vs C.precision** — if A > C by more than sampling noise, the LLM is identifying signal. If A ≈ C, the LLM might as well be a coin flip.
- **A.precision vs B.precision** — does adding the LLM improve the *quality* of buys, beyond what quant alone can do?
- **A.recall vs B.recall** — A will always have ≤ recall (it's a stricter filter); the question is whether the recall sacrificed was on losers (good filtering) or winners (over-filtering).
- **A's portfolio CAGR vs B's vs C's** — the bottom-line money question.

A clean **positive result** for the research question = `A > C` AND `A.portfolio_CAGR > SPY`. A clean **null result** = `A ≈ C` (the LLM isn't adding signal, modern LLMs can't yet automate Rule One qualitative judgement).
